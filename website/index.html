Summary

system_operating is an experimental minimal operating system for microcontroller devices. The focus is on establishing a low-level runtime foundation, with a clear path toward a simple shell, package management, and repository-backed application deployment.

Current progress

Completed

  • Core boot sequence and system initialization
  • Tonic Configuration System (.tnc) for declarative settings
  • Interactive Shell with comprehensive command support
  • Full Filesystem Integration (LittleFS) with unix-like utilities
  • Output handling with Serial and OLED display pagination
  • Modular Architecture (Core, Architecture, Extras)

Under development

  • Loom Package Manager for software distribution
  • Elin Bytecode format for custom user applications
  • Process management and non-preemptive multitasking
  • Improved display and input handling

Tonic Configuration (.tnc)

The system uses a declarative configuration format called Tonic. The configuration file is stored at /home/config.tnc.

system {
    hostname = "system-operating";
    version = "1.0.0";
}

network {
    ssid = "YourSSID";
    password = "YourPassword";
}

Loom & Elin — Future Roadmap

Overview

Loom is the planned package manager for system_operating. It is intended to manage packages compiled to Elin bytecode and to support a minimal distribution model for constrained hardware.

Planned capabilities

  • Manage Elin bytecode packages using a lightweight package registry
  • Update .tnc configuration files automatically to track installed packages and dependencies
  • Resolve package dependencies declared in .tnc files
  • Download and install packages from a web-based repository
  • Provide backup and recovery support through a central Loom repository portal

Intended workflow

  1. Write code locally in Elin and compile it to bytecode.
  2. Upload compiled bytecode to the Loom repository portal.
  3. Use Loom to fetch, install, and manage packages on the device.
  4. Maintain system state through .tnc configuration files.

Technical focus

Loom is designed around minimalism and efficiency. The goal is to support extremely limited environments, targeting small memory footprints and a declarative package model with a repository-backed backup strategy.
